# LAB 01 — Build a Mini-ISMS Pack (Risk → Controls → Policies)

## Duration
5 hours (in-class) + homework (2–3 hours)

## What you will produce (deliverables)
1. Asset Register
2. Risk Register
3. Risk Treatment Plan (top risks)
4. Control Mapping (organizational + technical)
5. Policy Pack (minimum viable policies)
6. Statement of Applicability (SoA-lite)

## Case study
See `handouts/CaseStudy-BlueRiverHealth.md`

## Tools
- Any spreadsheet editor (Excel/Google Sheets/LibreOffice)
- Markdown editor
- (Optional) Ubuntu VM for demonstrating technical controls

## In-class tasks
### Task 1 — Assets & context
- Build an asset register (min 15 assets).
- Identify asset owners and data types.

### Task 2 — Threats & risk scenarios
- Write 10 risk scenarios in plain language:
- "If <threat> exploits <vulnerability> then <impact> to/on <asset/business process>"
- Example: If Ransomware infection exploits Flat network architecture then imapct is High on Patient Database.

### Task 3 — Risk analysis
- Score likelihood and impact (define your scale).
- Prioritize risks (Low, Medium, High, Critical)

### Task 4 — Risk treatment & controls
- Choose treatment option for top risks (Mitigate, Transfer, Avoid, Accept)
- Select controls and describe how they reduce risk.

### Task 5 — Policy pack
Draft short policies:
- Access Control
- Backup & Recovery
- Incident Response
- Vendor/Supplier Security
- Acceptable Use

## Homework (submit report)
- Expand risk register to 12+ items.
- Complete treatment plan for top 5.
- Add at least 10 controls in mapping and justify each.
- Write a 1-2-page review with reflection from security perspective
