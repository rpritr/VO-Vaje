# Policy Pack (Minimum Viable Set)

## 1. Access Control Policy

- Disable shared accounts / Convert them into personal
- Enable MFA for critical systems and services
- Setup DDoS protection

## 2. Backup & Recovery Policy
- All critical systems must be backed up
- Daily incremental, Weekly full backups
- Backups must be stored on local secure storage (NAS) and in an offsite or cloud backup location
- Backup systems must be protected from unauthorized access, data must be encrypted
- Backup restoration tests must be conducted at least quarterly
- Backup retention must comply with healthcare regulatory requirements

## 3. Incident Response Policy

- Employees must report suspected security incidents immediately to the IT department
- Incident includes: Phishing attacks, Malware infections, Unauthorized access, Data breaches, Lost or stolen devices
- All incidents must be recorded in an incident log
- The IT Manager is responsible for coordinating incident response activities

## 4. Vendor Security Policy

- Vendors handling sensitive or patient data must undergo a security assessment before engagement
- All vendors processing personal data must sign a Data Processing Agreement (DPA)
- Vendors must demonstrate: data encryption, access control mechanisms, incident reporting procedures, compliance with relevant regulations
- Vendors must only be granted access necessary to deliver their services
- Vendor security performance must be reviewed periodically
- Vendor access must be revoked immediately when the contract ends

## 5. Acceptable Use Policy

- Company IT systems must only be used for business-related purposes
- Users must maintain strong passwords and must not share credentials
- Users must not install unauthorized software
- Patient information must only be accessed for legitimate medical purposes
- Patient data must not be stored on personal devices
- Devices must be locked when unattended
- Lost or stolen devices must be reported immediately
- Employees must exercise caution with email attachments and links
- Suspicious emails must be reported to IT
